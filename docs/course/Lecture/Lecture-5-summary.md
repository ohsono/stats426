## 1. Sequential Data & Time Series

- **Sequential data**: data where order matters: $X = \{x_1, x_2, ..., x_T\}$
- Types: numerical time series (financial, medical, sensor), text (discrete tokens), audio (continuous signals at ~16kHz), video (sequences of image frames)
- **Statistical objective**: estimate the conditional probability $P(x_t | x_{t-1}, ..., x_1)$

## 2. Stationarity

- **Strict stationarity**: the joint distribution of any subsequence is invariant to time shifts — very strong and hard to verify
- **Weak stationarity** (more practical):
  1. Constant mean: $E(x_t) = \mu$
  2. Constant finite variance: $Var(x_t) = \sigma^2$
  3. Covariance depends only on lag: $Cov(x_t, x_s) = \gamma(|t - s|)$
- **Autocovariance** $\gamma(h)$ and **autocorrelation** $\rho(h) = \gamma(h) / \gamma(0)$

## 3. Classical Autoregressive Models

### AR(1)
- $(x_t - \mu) = \phi * (x_{t-1} - \mu) + \epsilon_t$
- Weakly stationary when $|\phi| < 1$; explosive (non-stationary) when $|\phi| >= 1$

### AR(p)
- $x_t = \alpha + \sum_{j=1}^{p} \phi_j * x_{t-j} + \epsilon_t$
- Estimated via OLS by constructing lagged feature vectors
- Equivalent to a single-layer linear MLP (identity activation)

### Limitations of Linear Models
- Can only capture linear dependencies; underfits complex nonlinear relationships

## 4. MLP for Time Series (Nonlinear Autoregression)

- Replace linear combination with a feedforward neural network: $\hat y_t = f_{MLP}(y_{t-1}, ..., y_{t-p}; \theta)$
- **Pros**: universal approximation, learns lag interactions, minimal distributional assumptions
- **Cons**: fixed lookback window (lag p), data-hungry, black-box (loses interpretability of AR coefficients)

## 5. Probabilistic Modeling & the Markov Assumption

- Standard regression only models $E[x_t | history]$; the goal is the full conditional distribution $P(x_t | history)$
- **Markov assumption** makes this tractable:
  - First-order: $P(x_t | x_{t-1}, ..., x_1) ≈ P(x_t | x_{t-1})$
  - k-th order: condition on last k steps only
- AR(p) is a $p-th$ order Markov process

## 6. State Space Models (SSMs)

- Core idea: observations $y_t$ are generated by a latent (hidden) state $h_t$
- **Two equations**:
  - Transition: $h_t = A * h_{t-1} + B * u_t + \epsilon_t$
  - Observation: $y_t = C * h_t + \eta_t$
- **Linear SSM (LSSM)**: handles multivariate data; the Kalman Filter is the exact solution
- AR(1) maps to an SSM with h_t = x_t and perfect observation; AR(p) requires **state augmentation** (stacking p lags into a vector) to restore the first-order Markov property
- **Nonlinear SSM**: generalizes A, B, C to nonlinear functions f, g

## 7. From SSMs to RNNs

An RNN is a **nonlinear SSM** with two simplifications:
1. **Deterministic dynamics** — no process noise
2. **Neural parameterization** — functions approximated by learned weight matrices

$h_t = \sigma(W * h_{t-1} + U * x_t + b), y_t = g(h_t)$

## 8. The Vanilla RNN Cell

- Update equation: $h_t = \tanh(W_hh * h_{t-1} + W_xh * x_t + b)$
- **Why tanh?** Bounded output [-1, 1] prevents exploding activations that would occur with unbounded ReLU under repeated matrix multiplication
- Input $x_t$ can be: lagged output $y_{t-1}$ (pure autoregression), external covariates $u_t$, or a concatenation $[u_t, y_{t-1}]$

## 9. RNN Input-Output Architectures

| Pattern | Input | Output | Example |
|---|---|---|---|
| **One-to-One** | Single vector | Single vector | Image classification (MLP, not truly recurrent) |
| **One-to-Many** | Single seed | Sequence | Trajectory forecasting, music generation |
| **Many-to-One** | Sequence | Single value/label | Sentiment analysis, time series classification |
| **Many-to-Many** | Sequence | Aligned sequence | Video classification, named entity recognition |

- **Teacher forcing** (training): feed ground-truth $y_{t-1}$ as input
- **Autoregressive inference** (testing): feed predicted $\hat y_{t-1}$, which can cause error drift
- **Synchronous limitation**: input and output lengths must match; **Seq2Seq (Encoder-Decoder)** decouples them via a context vector

## 10. Vanishing & Exploding Gradients

- Gradient through time: $dL/dh_1$ is proportional to $(W_hh)^T$
- **Exploding** (eigenvalues $|\lambda| > 1$): gradients blow up to NaN; fix with **gradient clipping**
- **Vanishing** (eigenvalues $|\lambda| < 1$): gradients decay to zero, model forgets early inputs; fix with **LSTM / GRU / ResNets**

## 11. Deep (Stacked) RNNs

- Stack multiple RNN layers; output of layer l-1 becomes input to layer l
- Learns hierarchical representations: raw patterns -> composed patterns -> high-level abstractions

## 12. LSTM (Long Short-Term Memory)

Addresses the vanishing gradient problem by introducing **dual states**:
- **Cell state $C_t$**: long-term "conveyor belt" memory with only linear interactions
- **Hidden state $H_t$**: short-term, filtered output exposed to the next layer

### Three Gates (all sigmoid-based, values in (0,1)):
1. **Forget gate $F_t$**: how much old memory to keep
2. **Input gate $I_t$**: how much new information to write
3. **Output gate $O_t$**: how much internal memory to reveal

### Update equations:
- Candidate: $\tilde{C}= \tanh(W_xc * x_t + W_hc * H_{t-1} + b_c)$
- Cell update: $C_t = F_t \cdot C_{t-1} + I_t . \tilde{C}$
- Hidden state: $H_t = O_t \cdot \tanh(C_t)$

**Key insight**: when $F_t ≈ 1$ and $I_t ≈ 0$, the cell state passes through unchanged, preserving gradients over long distances.

## 13. GRU (Gated Recurrent Unit)

A simplified alternative to LSTM (Cho et al., 2014):
- **Single hidden state** h_t (no separate cell state)
- **Two gates** instead of three:
  1. **Reset gate $R_t$**: controls how much past state to ignore when computing the candidate
  2. **Update gate $Z_t$**: acts as combined input/forget gate

### Update equations:
- Candidate: $\tilde{h} = \tanh(W_xh * x_t + W_hh * (R_t \cdot h_{t-1}) + b_h)$
- Hidden state: $h_t = Z_t \cdot h_{t-1} + (1 - Z_t) \cdot \tilde{h}$

## 14. Comparison: RNN vs. LSTM vs. GRU

| Feature | Vanilla RNN | LSTM | GRU |
|---|---|---|---|
| Memory unit | $h_t$ only | $C_t$ (cell) + $H_t$ | $h_t$ only |
| Gates | 0 | 3 (input, forget, output) | 2 (reset, update) |
| Parameters | 1x weights | 4x weights | 3x weights |
| Gradient flow | Vanishing | Preserved (identity path in $C_t$) | Preserved (interpolation) |
| Tradeoff | Fast but forgets | Powerful, standard for complex tasks | Efficient, comparable performance |
