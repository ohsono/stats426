Forward: The RNN unrolls over time stepst= 1. . .T, computing hidden

Practical Solution: Truncated BPTT
Full BPTT is often computationally infeasible for very long sequences (e.g., long
time series data, or long text passages) due to memory constraints and gradient
instability.
The Strategy
1.Splitthe long sequence into shorter subsequences of lengthτ(e.g.,τ≈35).
2.Detachthe computational graph at the subsequence boundaries.
Mechanism:
Forward Pass: The final hidden stateh τ of blockkbecomes the initial state
h0 of blockk+ 1. This preserves thecontext.
Backward Pass: We stop the gradient flow at the boundary ( ∂L
∂hτ
←0).
Trade-off:This limits the gradient’s ”memory” toτsteps, sacrificing long-term
dependency learning for computational efficiency and stability.

Training RNNs
Summary of Training Dynamics
1.BPTTis standard backpropagation applied to an unrolled time graph.
2.Shared Weights:The sameW hh is used at every step, meaning gradients
from all steps accumulate into one update forW hh.
3.Stability:The recursive multiplication of matrices makes RNNs inherently
unstable (Vanishing/Exploding gradients).
4.Truncation:We typically truncate the backward pass afterτsteps to save
memory and prevent divergence, treating the initial hidden state of the block
as a constant.

Deep RNNs
Motivation for Deep RNNs
Standard RNNs typically have asingle hidden layer.
This implies only one linear transformation + activation per time step.
Limitation: It is difficult to capture highly non-linear, hierarchical features
(e.g., syntax→semantics→sentiment).
Going deep:
Just as Deep CNNs stack layers to separate features (edges→shapes→
objects), Deep RNNs stack multiple recurrent hidden layers.
Benefit: Allows the network to process information at different time scales
and levels of abstraction simultaneously.

Implementation & Summary
Computational Cost: Training Deep RNNs requires backpropagation along two
axes:
1. Through Time: Gradients flow backward along the unrolled time dimension
(often truncated via Truncated BPTT).
2. Through Depth: Gradients flow down through stacked layers (from layerLto
1).
Scaling:Costs increase with sequence lengthT, batch sizeB, hidden sizeH, and depth
L. Memory usage is dominated by storing activations at every steptand layerlfor the
backward pass.
Notes on implementation:
While libraries handle the autograd math and providenum layers, the data scientist
must explicitly manage:
Truncation:Deciding the window sizeτfor BPTT.
Variable Lengths:Usingpack padded sequenceto mask padding tokens efficiently.
State Management: Detachinghidden states between batches (‘h.detach()‘) to
prevent the graph from growing infinitely and consuming all memory.

The Limitations of Standard RNNs
So far, we have discussed building simple/deep RNNs (LSTM/GRU) for
one-to-many, one-to-one or many-to-one tasks.
However, in In standard RNN formulations, the input sequence lengthT x and
output sequence lengthT y usually share a rigid relationship (oftenT x =T y or
Ty = 1).
The Problem (Sequence-to-Sequence):
Natural Language Processing:
▶ Source: ”Hello world” (T x = 2)
▶ Target: ”Bonjour le monde” (T y = 3)
Time Series Forecasting:
▶ Input: Past 24 hours of sensor data (T x = 24)
▶ Output: Future 48 hours of predictions (T y = 48)
Here,T x ̸=T y , and the alignment is non-monotonic. An MLP cannot handle this
because the input dimension is variable. A standard Many-to-Many RNN can not
handle this because the output length is not knowna priori.

Encoder-Decoder Architectures
The Encoder-Decoder Paradigm
To solve the variable-length mapping problem, two distinct sub-networks are
introduced and jointly trained:
1. The Encoder: A functionf enc that reads the variable-length input sequence
x1,· · ·,x Tx and compresses it into a fixed-length internal representation.
2. The Context Variable (c): The fixed-length vector (or set of vectors) that
encapsulates theentiresemantic information of the input sequence. This is
the information bottleneck.
3. The Decoder: A functionf dec that takes the context variablecand generates
the variable-length output sequencey 1,· · ·,y Ty , one token at a time.

Summary and The Bottleneck Problem
Summary of Mechanisms
1. Encoding:x 1...T →RNN→h T →c.
2. Decoding: (c,y <t′)→RNN→s t′ →P(y t′).
3. Loss: Cross-Entropy/ℓ 2 summed over output sequence lengthT ′.
The Theoretical Limitation: The Context Variablecis a fixed-size vector. It must
encode the nuance of a sentence regardless of whether the sentence has 5 words
or 100 words, or the time series is long or short.
This creates an Information Bottleneck.
AsTincreases, the ability ofh T to retain information fromx 1 (the start of the
sequence) diminishes due to the fixed capacity and vanishing gradients. This
limitation necessitates the Attention Mechanism (future topic).

Breaking the Context Bottleneck
Recap: The Encoder-Decoder Challenge: In standard Seq2Seq, the Encoder must
compress theentiresource sequence into a single fixed-length vectorc.
The Problem:
Information Loss:For long sequences (e.g., many words, observations),c
acts as a bottleneck, forgetting early details.
Path Length:To access information fromx 1 at the end of generation, the
gradient must flow back throughTtime steps (vanishing gradient risk).
The Solution (Attention): Instead of a staticc, we generate a dynamic context
vectorc t′ foreverydecoding stept ′.
This creates Direct Connections between the Decoder state and specific Encoder
states, reducing the path length fromO(T) toO(1).

Attention in the Encoder-Decoder Loop
How does this all fit into the translation model we started with?
1.The Setup (Encoder):The Encoder processes source words (”Hello”,
”World”) into hidden statesh 1,h 2.⇒These become the Keys (K) and
Values (V).
2.The Trigger (Decoder Stept ′):The Decoder has just generated ”Bonjour”
