The Transformer Architecture
Motivation: Why Transformers?
Prior State of the Art (RNNs/LSTMs):
▶ Strictly sequential computation:h t =f(h t−1 ,x t).
▶ Precludes parallelization within training examples (a bottleneck for modern
GPUs).
▶ Long-range dependencies are notoriously hard to learn due to information
decay over time.
The Transformer Solution (Vaswani et al., 2017):
▶ Eschew recurrence entirely.
▶ Rely solely on attention mechanisms to draw global dependencies, creating
paths of lengthO(1) between any two tokens in a sequence.
▶ ”Attention Is All You Need.”

What it does: Calculates the dot product between every Query and every Key.

Result: Yields a matrix of raw, unnormalized ”attention scores” representing
the alignment between all tokens.

Recap: Attention Mechanisms
Attention Breakdown: Scale and Mask
Scale (÷ √dk):
▶ As dimensions (d k) grow, dot products naturally produce larger magnitude
values.
▶ Large values push the Softmax function into regions with extremely small
gradients (vanishing gradients).
▶ Dividing by the square root of the dimension stabilizes the variance and
ensures healthy gradient flow.
Mask (Optional):
▶ Used primarily in the Decoder (or for padded tokens).
▶ Replaces specific alignment scores with−∞.
▶ Ensures the model cannot ”look into the future” during autoregressive
generation, preserving the causal flow of time.

Recap: Attention Mechanisms
Attention Breakdown: Softmax and Final MatMul
Softmax:
▶ Converts the raw, scaled scores into a probability distribution.
▶ Applied row-wise: For each Query (row), the attention weights distributed over
all Keys (columns) sum exactly to 1.0.
▶ Tokens with scores of−∞(from the Mask) become exactly 0.
Final MatMul (×V):
▶ We multiply the resulting probability weights by the Values (V).
▶ Intuition: We take aweighted sumof the actual data. If a specific token’s Key
had a 90% match with our Query, its Value will make up 90% of the final
output vector for that position. Irrelevant tokens are effectively filtered out.

Positional Encoding
The ”Bag of Words” Problem: Recall that Multi-Head Attention operates on
all tokens simultaneously. Without explicit sequence data, the model would
not know if a word is at the beginning or end of a sentence.
The Injection Point: Positional Encoding happens before the sequence enters
the first Encoder block.
The Mechanism:
▶ We take the sequence of token Embeddings (E).
▶ We generate a geometric Positional Encoding matrix (PE) of the exact same
dimensions using sine and cosine waves.
▶ The Input to the first block is simply:X=E+PE.
Subsequent Layers: For Blocks 2 throughN, the position data does not need
to be re-injected; it is preserved and passed upward via the residual
connections.

The Add & Norm Block - I
Every sub-layer (Self-Attention or FFN) is wrapped in a residual connection and
layer normalization.
The ”Add” Residual Connection
The Problem:Deep networks suffer from vanishing gradients as the chain
rule multiplies small numbers during backpropagation.
The Solution:Adding the input back to the output (x+ Sublayer(x))
creates a linear bypass. Gradients flow backwards completely unchanged,
allowing us to stack dozens of layers.
Requirement:To addxto Sublayer(x), both must have the exact same
dimension (d model = 512).
Architectural Evolution: Post-LN vs. Pre-LN
Original (Post-LN):Output = LayerNorm(x+ Sublayer(x))
Modern (Pre-LN):Output =x+ Sublayer(LayerNorm(x))
Why the change?Models like GPT and LLaMA use Pre-LN because it leaves
the residual highway (x+. . .) completely uninterrupted by normalization
layers. This dramatically improves training stability for very deep networks.

The Feed-Forward Network (FFN)
If Self-Attention is how tokens ”talk” to each other, the FFN is how each token
”processes” what it learned from the conversation.
”Position-wise”: The FFN is applied to each token’s positioniindependently
but identically (using the exact same shared weightsW 1,W 2).
The Expansion-Compression Operation
FFN(x) = Activation(xW 1 +b 1)W2 +b 2
Step 1 (Expand):Project thed model-dimensional token (e.g., 512) up to a
much wider hidden dimension (e.g.,d ff = 2048).
Step 2 (Non-Linearity):Apply an activation function (like ReLU or GELU).
Self-Attention is entirely linear; this step is crucial for learning complex,
non-linear representations.
Step 3 (Compress):Project back down tod model = 512 so the vector can be
seamlessly added back into the residual stream.

Digression: Activation Functions (ReLU vs. GELU)
While the original Transformer used ReLU, modern architectures (BERT, LLaMA,
GPT) overwhelmingly rely on GELU or its variants (like SwiGLU).
ReLU (Rectified Linear Unit)
Formula:f(x) = max(0,x)
Behavior:Strict gating. Ifx<0,
the output is exactly 0 (a hard
switch).
Pros:Computationally extremely
cheap.
Cons:”Dead Neurons.” If weights
push a unit deeply negative, its
gradient becomes zero and it
stops learning entirely.
GELU (Gaussian Error Linear Unit)
Formula:f(x) =x·Φ(x)
(Φ(x) is the CDF of the standard normal
distribution)
Behavior:Probabilistic gating. Weights inputs by
their value, modulated by the probability that the
input is greater than other randomly drawn inputs.
Advantage:Continuously differentiable (smooth)
everywhere. Allows a small amount of negative
info to flow, creating a smoother optimization
landscape.

Why Positional Encoding?
The Transformer architecture contains no recurrence and no convolution.
Without positional information, the model is entirely permutation invariant.
The Problem: ”Dog bites Man” would look mathematically identical to
”Man bites Dog” to the self-attention mechanism (acting simply as a parallel
bag-of-words).
The Solution: We must explicitly inject structural information about the
relative or absolute position of tokens. We do this by adding a ”positional
vector” to the input embeddings at the very bottom of the stack.

Evolution of Positional Encodings: Context Extrapolation - II
1. Learned Absolute Embeddings (BERT, GPT-2/3)
Instead of sine waves, the model simply learns a unique, trainable vector for
pos= 1,pos= 2, . . . ,pos=N.
Extrapolation:None. If trained on 2048 tokens, it physically cannot process
token 2049 because no embedding exists for that position.
2. RoPE: Rotary Positional Embedding (LLaMA, Mistral)
Injects positional info directly into the Queries (Q) and Keys (K) at each
layer by applying a mathematical rotation in the complex plane.
The dot productQ m ·K n becomes a function only of the relative distance
(m−n).
Extrapolation:Good. Can be extended to massive contexts via ”RoPE
scaling” (interpolating the rotation angles) with minimal fine-tuning.
3. ALiBi: Attention with Linear Biases (BLOOM)
Discards input positional embeddings entirely. Instead, it subtracts a static,
distance-based penalty directly from the pre-Softmax attention scores:
score(qi ,k j)−m· |i−j|.
Extrapolation:Excellent. A model trained on 1K tokens can zero-shot
evaluate on 10K+ tokens gracefully because it inherently penalizes distant

Remarks on Computing and Memory Complexity
While Multi-Head Attention is highly parallelizable, its standard dense
implementation scales poorly with the sequence length (T).
1. Time Complexity:O(T 2 ·d)
To compute the attention scores, we multiply the Query matrixQ∈R T×d by
the Key matrixK ⊤ ∈R d×T (whereTis the number of tokens, anddis the
embedding dimension).
Computing thisT×Tsimilarity matrix requiresT 2 dot products, each of
dimensiond.
2. Memory Complexity:O(T 2)
The resultingT×Tattention weight matrix must be stored in memory to
compute the Softmax and gradients.
The Scaling Problem: Doubling the context windowquadruplesthe memory
and compute requirements.
Example:Scaling from aT= 2,000 toT= 100,000 token sequence
increases the attention matrix footprint by 2,500×per layer, per head.

Solution to the Computational and Memory Complexity Issue
1. Hardware Optimization: FlashAttention
Standard implementations physically write the massiveT×Tmatrix to slow
GPU High-Bandwidth Memory (HBM).
FlashAttention(Dao et al., 2022) uses hardware-aware ”tiling” to compute
the exact same mathematical result entirely in fast on-chip SRAM, bypassing
the HBM read/write bottleneck.
2. Architectural Alternatives: Mamba & SSMs
Rather than optimizing theO(T 2) attention math,Mamba(Gu & Dao,
2023) replaces it entirely with Selective State Space Models (SSMs).
It compresses history into a hidden state, achievingtrue linearO(T)scaling
in training andO(1) memory during generation, offering a strong alternative
to Transformers for ultra-long sequences.

Deep Dive: ”Outputs (Shifted Right)” & Teacher Forcing
Speeding up Training
The Decoder is strictly auto-regressive. It consumes its own previous outputs to
generate the next one.
Looking at a language example:
Step 1:Feed the model<SOS>(Start of Sequence). It predicts:"The"
Step 2:Feed the model<SOS>,"The". It predicts:"cat"
Step 3:Feed the model<SOS>,"The","cat". It predicts:"sleeps"
Key issue: If we train the model this way, it is incredibly slow (we lose all parallel
computing power). Furthermore, if the model guesses wrong early on, the errors
compound, making it impossible to learn later words effectively.

Teacher Forcing

Training Step 4: Backpropagation & The Optimizer
Transformer training can be unstable early on due to high gradient variance and
deep residual signal propagation.
Adam / Adam-W Optimizer:Transformers almost universally rely on Adam
or Adam-W, which maintain adaptive, per-parameter learning rates via
moving averages of gradients and squared gradients.
Learning Rate Warmup:A critical training trick. The learning rate is
gradually increased (typically linearly) from a very small value over the first
few thousand steps.
Why?Early in training, attention patterns are essentially random. Large
parameter updates during this phase can destabilize representation learning.
Warmup limits update magnitude until attention mechanisms and internal
representations begin to organize.

Remarks on the Training Pipeline
Modern Transformers are notoriously sensitive to initialization and optimization
choices. The following “recipe” is used across BERT, GPT, and LLaMA families.
1. Optimization & Learning Rate
Optimizer (AdamW): Standard Adam can fail on Transformers. AdamW
decouples weight decay from the adaptive gradient update, improving
generalization.
LR Schedule (Warmup + Cosine Decay):
▶ Warmup:Linearly increase the LR from 0 for the first 1–5% of steps. This
prevents early divergence caused by the high variance of uninitialized
LayerNorms and deep residual paths.
▶ Decay:Smoothly anneal the LR down to∼10% of its peak value using a
cosine curve as the model converges.
2. Regularization & Stability
Gradient Clipping: Cap the global norm of gradients (usually to 1.0) to
prevent exploding gradients during unexpected training loss spikes.
Weight Decay: Typically set around 0.01 to 0.1, but crucially, it isnever
applied to 1D parameters (biases and LayerNorm scales/shifts).

Inference Step 3: The KV Cache (Optimization)
Naive decoding is inefficient.
Without caching, at timestept, the model recomputes attention projections for all
previous tokens.
KV Caching
Store Keys and Values from previous steps in memory.
At step t, compute K/V only for the newly generated token and append it to
the cache.
The new token attends to these cached representations.
Impact:
Avoids redundant computation.
Reduces inference latency dramatically.
Essential for large-scale LLM deployment.
Note: Each new token still attends to prior tokens (linear in context length), but
we avoid recomputing the full prefix each step.
